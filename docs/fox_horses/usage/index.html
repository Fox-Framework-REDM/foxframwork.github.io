Usage and API

Installation, configuration, commands, exports, and database notes for fox_horses.

Installation

Import sql/install.sql into the same database used by fox_core, then ensure the resource after its dependencies.

Configuration

Edit config.lua and shared/config.lua.

  • Config.Stables: stable locations, labels, spawn points, previews, and blips
  • Config.MaxHorses: horse slot limit per character
  • Config.MoneyItem: inventory item used as money
  • Config.Purchase: horse purchase settings and currency items
  • Config.Sale / Config.Transfer: sale refunds and transfer distance
  • Config.Care: care item requirements, action duration, and status deltas
  • Config.Equipment: equipment pricing, distance, and tint settings
  • Config.Bag: saddlebag slots, weight, and stash settings
  • Config.Water: trough and river drinking validation distances
  • Config.Prompts: active horse prompt keys and prompt layers
  • Config.Whistle: whistle command, key, distance, and cooldown settings
  • HorseConfig.Status: starting horse status values

Web Build

The NUI is served from web/dist/index.html.

Player Commands

  • /horses: opens the default stable menu.
  • /horsebag: opens the saddlebag for the mounted horse.
  • /horsestatus: shows health, status, and saddlebag data for the mounted horse.
  • /horsegearrefresh: reapplies visual equipment for the mounted horse.
  • /horseloyalty: shows loyalty and bond data for the mounted horse.
  • /fox_horses_whistle: whistles for the primary horse. The key is configured by Config.Whistle.key.

Server Exports

GetHorse(vehicleId)

Returns the horse row from the canonical fox_core vehicle table.

GetOwnedHorses(charId)

Returns all horses owned by a character.

IsHorseOwnedBy(vehicleId, charId)

Returns whether a character owns the horse.

GetHorseEquipment(vehicleId)

Returns the persisted equipment table for the horse.

GetHorseStatus(vehicleId)

Returns the persisted status row for the horse.

GetHorseLoyalty(horseId, characterId)

Returns loyalty and bond data for a horse and character.

AddHorseLoyalty(horseId, characterId, amount, reason?)

Adds loyalty XP server-side.

RemoveHorseLoyalty(horseId, characterId, amount, reason?)

Removes loyalty XP server-side.

CanControlHorse(horseId, characterId)

Returns whether a character can control a horse based on loyalty rules.

GetHorseCommandModifiers(horseId, characterId)

Returns command modifiers derived from loyalty.

GetHorseFavoriteOwner(horseId)

Returns the horse favorite owner data.

GetBagCapacity(vehicleId)

Returns computed saddlebag capacity.

Database

fox_horses keeps the canonical horse and wagon ownership record in fox_core.vehicles.

sql/install.sql adds extension tables for gameplay data such as:

  • horse profiles and status
  • training and breeding
  • genetics and lineage
  • loyalty and ownership history
  • cooldowns and logs

Security Notes

  • The client only sends intent.
  • The server resolves the active character through fox_core.
  • Ownership, type, distance, entity, net id, cooldowns, and locks are validated server-side.
  • Inventory changes go through ox_inventory.
  • SQL uses oxmysql with parameterized queries.
  • Horse and wagon state is persisted through fox_core plus fox_horses extension tables.
