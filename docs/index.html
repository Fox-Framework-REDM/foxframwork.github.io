Introduction

RedM framework (ox_core port) for rdr3 servers.

fox_core is a framework for RedM (Red Dead Redemption 2 / rdr3), built on the architecture of overextended's ox_core, but natively ported to RDR3's natives and limitations — no cars, armour, plates or phones; horses, cores, and telegrams instead.

Where to start

  • fox_core — players, characters, accounts and vehicles/horses.
  • fox_clothing — RedM clothing stores and wardrobes.
  • fox_interact — RedM world interaction UI and exports.
  • fox_charselect — multicharacter and spawn selector.
  • fox_lib — shared RedM utility helpers.
  • fox_hud — HUD, UI state locks, progress, announcements and menu base.

Conventions

  • Network events keep the ox: namespace for contract stability, even though the resource is fox_core.
  • Exports are accessed via exports.fox_core (Ox = exports.fox_core).
  • Convars use the ox: prefix (e.g. ox:spawnModel).
